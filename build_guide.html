Publishing Guide

Instructions for generating signed APK and AAB files in Visual Studio

Note: Run these commands in the Package Manager Console (Tools > NuGet Package Manager > Package Manager Console) or PowerShell within Visual Studio.

Phase 1: Keystore Generation

1

If you don't have a keystore, create a new one using the following command. replace the placeholders with your desired values.

&"C:\Program Files\Android\openjdk\jdk-21.0.8\bin\keytool.exe" -genkeypair -v -keystore varnam_production.keystore -alias varnam_key -keyalg RSA -keysize 2048 -validity 10000

* This will prompt you for passwords and organization details.

Phase 2: Building the Signed App

2

To generate a signed APK:

dotnet publish -f net10.0-android -c Release -p:AndroidKeyStore=true -p:AndroidSigningKeyStore=varnam_production.keystore -p:AndroidSigningStorePass=YOUR_PASSWORD -p:AndroidSigningKeyAlias=varnam_key -p:AndroidSigningKeyPass=YOUR_PASSWORD
3

To generate a signed AAB (for Play Store):

dotnet publish -f net10.0-android -c Release -p:AndroidKeyStore=true -p:AndroidSigningKeyStore=varnam_production.keystore -p:AndroidSigningStorePass=YOUR_PASSWORD -p:AndroidSigningKeyAlias=varnam_key -p:AndroidSigningKeyPass=YOUR_PASSWORD -p:AndroidPackageFormat=aab

Phase 3: Locating Your Files

4

After the command finishes, your files will be located in:

Varnam\Varnam\bin\Release\net10.0-android\publish\
Important: Keep your .keystore file and passwords safe. You will need the exact same key to update your app on the Play Store in the future.
